如何使用Hexo搭建个人博客

准备工作

  • 安装git
  • 安装nodejs
  • 申请github账号
  • (可选)自己的域名

hexo环境

安装Hexo

搭建博客

初始化博客

初始化博客完成后，会看到一大串命令行，然后最后一句话是Start blogging with Hexo!，并且默认使用经典主题landscape，此时博客已经建立好，可以在博客根目录下运行一下命令查看

配置博客信息

配置站点信息

找到根目录下_config.yml文件，使用VsCode打开

Site
Title 网站标题
Subtitle 网站副标题
description 描述，一般网站可用于SEO优化，部署在github pages上就没什么用了
author 作者，可以填写你的姓名，外号，昵称等等
language 语言，默认是英文，中文: zh-CN
timezone 时区，默认是使用你电脑上对应的时区，不喜欢自己可以设置成UTC
URL
url 网站URL
root 网站根目录，使用默认 / 即可
permalink 点击详情页的时候链接标识符，使用默认即可
permalink_defaults

到这里，基本网站信息配置就已经完工，可以运行hexo server 来查看

发布到网上

使用github pages

首先申请一个github账号，然后新建Repository，名字和你的用户名一样，具体原因不细说，可以搜一下

生成静态文件

安装插件

安装 hexo-deployer-git

配置发布地址

找到根目录下_config.yml里边的deploy节点

github
type 发布网上必填，部署类型，选择git
repo 发布网上必填，仓储地址，你的github pages仓储地址
branch 分支，仓储分支，可不填
message 发布信息，可不填

hexo不仅仅可以部署到github

发布到github

耐心等待，发布完成后，使用http://github用户名.github.io 即可查看发布后的结果

自定义github pages域名

github pages设置

在仓库中找到settings，然后看到custom domain，即自定义域名，填写自己要部署的域名，如http://hexo.asiacream.cn,然后再到你的域名解析中心，新建一个CNAME类型的解析，记录地址使用 github用户名.github.io，然后就可以使用自定义域名进行访问了

新建文章

这样就会在source/_posts中生成一个“文章名”.md的文件，点击进去，有title和创建时间，随便填写一些东西，使用hexo g 生成静态文件，然后部署到github中，接着使用自定义域名，会发现此时自定义域名又不可用了

自定义域名设置

找到自定义域名，发现发布之后刚刚填写自定义域名没有了，重新填写好自定义域名后浏览器中打开，又可用了，那就出现一个问题，不能每次部署后都到github上修改一次配置吧，需要想办法解决

解决办法

找到根目录下source，点击进入文件夹，新建文件，CNAME，注意，没有后缀名，把自定义域名填写上，如hexo.asiacream.cn，接着重新部署到github上，发现刚刚的问题解决了
